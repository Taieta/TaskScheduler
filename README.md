# TaskScheduler

Необходимо разработать библиотеку **TaskScheduler**, предназначенную для построения и выполнения графа зависимых вычислений.

Во многих задачах отдельные вычисления зависят от результатов других вычислений. Такие зависимости удобно представлять в виде **графа задач**, где:

* вершина графа — отдельная вычислительная задача;
* ребро графа — зависимость между результатом одной задачи и аргументом другой.

Библиотека должна позволять регистрировать задачи, автоматически учитывать зависимости между ними и обеспечивать корректный порядок их выполнения. При этом каждая задача должна выполняться **не более одного раза**, даже если её результат используется несколькими последующими вычислениями.

В качестве примера рассмотрим вычисление корней квадратного уравнения

$$
ax^2 + bx + c = 0,
$$

где гарантируется существование двух действительных корней.

При непосредственной реализации дискриминант обычно вычисляется дважды — отдельно для каждого корня. Использование **TaskScheduler** позволяет вычислить дискриминант и его квадратный корень только один раз, а затем использовать полученный результат во всех зависимых задачах.

Пример использования библиотеки:

```cpp
float a = 1;
float b = -2;
float c = 0;

TaskScheduler scheduler;

auto id1 = scheduler.add(
    [](float a, float c) { return -4 * a * c; },
    a, c
);

auto id2 = scheduler.add(
    [](float b, float value) { return b * b + value; },
    b, scheduler.getFutureResult<float>(id1)
);

auto id3 = scheduler.add(
    [](float b, float d) { return -b + std::sqrt(d); },
    b, scheduler.getFutureResult<float>(id2)
);

auto id4 = scheduler.add(
    [](float b, float d) { return -b - std::sqrt(d); },
    b, scheduler.getFutureResult<float>(id2)
);

auto id5 = scheduler.add(
    [](float a, float value) { return value / (2 * a); },
    a, scheduler.getFutureResult<float>(id3)
);

auto id6 = scheduler.add(
    [](float a, float value) { return value / (2 * a); },
    a, scheduler.getFutureResult<float>(id4)
);

scheduler.executeAll();

std::cout << "x1 = " << scheduler.getResult<float>(id5) << std::endl;
std::cout << "x2 = " << scheduler.getResult<float>(id6) << std::endl;
```

В приведённом примере задачи образуют ориентированный ациклический граф зависимостей. Метод `executeAll()` должен определить корректный порядок выполнения задач и вычислить результат каждой из них только один раз.

Метод `getFutureResult()` возвращает объект, представляющий **будущий результат** задачи. Такой объект может быть передан в качестве аргумента при создании других задач, что позволяет явно задавать зависимости между вычислениями без необходимости предварительного выполнения исходной задачи.

### Публичный интефейс TTaskScheduler

 - **add** - принимает в качестве аргумента задание для него. Возвращает объект описывающий добавленную таску.
 - **getFutureResult<T>** - возвращает объект, из которого в будущем можно получить результат задания, переданного в качестве результата типа Т
 - **getResult<T>** - возвращает результат выполнения задания определенного типа. Вычисляет его если оно еще не подсчитано
 - **executeAll** - выполняет все запланированные задания

### Требования и ограничения к заданиям

  - [Callable object](https://en.cppreference.com/w/cpp/named_req/Callable)
  - Аргументы гарантированно [CopyConstructible](https://en.cppreference.com/w/cpp/named_req/CopyConstructible)
  - Возвращаемое значение произвольное, гарантированно [CopyConstructible](https://en.cppreference.com/w/cpp/named_req/CopyConstructible)
  - Количество аргументов не больше 2
